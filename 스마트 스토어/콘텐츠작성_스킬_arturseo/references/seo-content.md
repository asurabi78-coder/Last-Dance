# SEO Writing Best Practices for Content Creators

Guidelines for creating content that ranks in search engines while remaining valuable and readable.

---

## Keyword Strategy

### Research Before Writing
- Identify one **primary keyword** per page (the main search query you want to rank for)
- Find 3–5 **secondary keywords** (related terms, synonyms, long-tail variations)
- Check **search intent**: Is the searcher looking to learn, compare, buy, or navigate?
- Match content format to intent:
  - Informational → How-to guide, explainer, listicle
  - Comparison → "X vs Y", "Best alternatives to X"
  - Transactional → Landing page, product page, pricing page
  - Navigational → Brand page, documentation

### Keyword Placement
- **Title / H1**: Include primary keyword naturally (front-load when possible)
- **URL slug**: Short, keyword-rich, no dates (e.g., `/seo-writing-guide` not `/2026/03/guide-to-seo-writing-tips-2026`)
- **First 100 words**: Use primary keyword in the opening paragraph
- **H2 headings**: Include secondary keywords in at least 2 subheadings
- **Meta description**: 150–160 characters, include primary keyword + benefit + action verb
- **Image alt text**: Describe the image accurately; include keyword only when natural
- **Body text**: Use keywords naturally — never force them. Aim for 0.5–1.5% keyword density (roughly).

## Content Structure for SEO

### Page Layout
```
Title (H1) — one per page, includes primary keyword
├── Introduction (100–200 words, includes keyword)
├── H2: Key Section 1 (secondary keyword)
│   ├── H3: Subsection
│   └── H3: Subsection
├── H2: Key Section 2 (secondary keyword)
├── H2: Key Section 3
├── H2: FAQ or Common Questions (featured snippet targets)
└── Conclusion / Key Takeaways + CTA
```

### Formatting Best Practices
- **H2 every 200–300 words** — search engines use headings to understand structure
- **Bullet points and numbered lists** — improve readability and win featured snippets
- **Short paragraphs** — 2–4 sentences max for digital content
- **Bold key phrases** — helps scanners and signals importance
- **Table of contents** — for posts over 1,500 words (helps with sitelinks)
- **Images every 300–500 words** — break up text, add alt text, compress for speed

## Meta Descriptions

### Formula
```
[Primary keyword] + [What the reader gets] + [CTA or hook]
```

### Examples
- "Learn 12 SEO writing techniques that drive organic traffic. Step-by-step guide with templates."
- "Compare the top 5 project management tools for remote teams. Find your best fit in 5 minutes."

### Rules
- 150–160 characters (Google truncates beyond this)
- Include the primary keyword (it gets bolded in search results)
- Unique per page — never duplicate meta descriptions
- Write like ad copy — this is your search result pitch

## Internal Linking

### Why It Matters
- Distributes page authority across your site
- Helps search engines discover and understand content hierarchy
- Keeps readers on your site longer (reduces bounce rate)
- Establishes topical clusters

### Best Practices
- **2–5 internal links per post** (more for long-form content)
- **Descriptive anchor text**: Use "SEO writing guide" not "click here" or "this article"
- **Link to relevant content**: Only link where it genuinely helps the reader
- **Update old posts**: When you publish new content, go back and add internal links from existing related posts
- **Hub and spoke model**: Create pillar pages (broad topic) that link to cluster pages (subtopics), and vice versa

---

## External Linking

- **1–3 authoritative external links per post** — signals credibility
- Link to primary sources: research papers, official documentation, industry reports
- Avoid linking to direct competitors' commercial pages
- Open external links in new tabs (standard UX for outbound links)

## Featured Snippets

### How to Target Them
Featured snippets appear above result #1 — massive visibility boost.

- **Paragraph snippets**: Answer a question directly in 40–60 words right after an H2 that asks the question
- **List snippets**: Use H2 as the question, followed by a numbered or bulleted list
- **Table snippets**: Use HTML tables for comparison data
- **Definition snippets**: "What is [term]?" → concise 1–2 sentence definition

### Example
```markdown
## What Is Content Repurposing?

Content repurposing is the process of adapting a single piece of content into multiple
formats for different platforms. For example, turning a blog post into a LinkedIn post,
Twitter thread, and email newsletter to maximize reach without creating from scratch.
```

## Content Depth & Quality Signals

### E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness)
- **Experience**: Include first-hand examples, case studies, original data
- **Expertise**: Demonstrate deep knowledge — don't skim the surface
- **Authoritativeness**: Cite credible sources, link to research, show credentials
- **Trustworthiness**: Be accurate, transparent about limitations, include dates

### Depth Checklist
- [ ] Does this answer the query better than the current top 3 results?
- [ ] Are there subtopics competitors miss that I've covered?
- [ ] Do I include original examples, data, or insights?
- [ ] Is every claim supported with evidence or experience?
- [ ] Would a subject-matter expert approve this content?

## Technical SEO for Content Creators

You don't need to be a developer, but these matter:

- **Page speed**: Compress images (WebP format), avoid huge embedded videos
- **Mobile-friendly**: Test that your content reads well on mobile
- **URL structure**: Short, descriptive, lowercase, hyphens (not underscores)
- **Schema markup**: FAQ schema for FAQ sections, HowTo schema for tutorials, Article schema for blog posts
- **Canonical tags**: One URL per piece of content (avoid duplicate content issues)
- **Title tag length**: 50–60 characters for full display in search results

## Content Update Strategy

Search engines favor fresh, accurate content.

- **Quarterly review**: Check top-performing pages for outdated stats, broken links, or new information
- **Annual refresh**: Update titles, add new sections, refresh examples for evergreen content
- **Date handling**: Update "last updated" dates when making substantial changes
- **Redirect old URLs**: If you change a slug, 301 redirect the old URL
- **Consolidate thin pages**: Merge underperforming similar pages into one comprehensive piece
