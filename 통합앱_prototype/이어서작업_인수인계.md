## 0. 가장 중요한 규칙 (반드시 지킬 것)
- **모든 변경은 계획 → 검증 → 적용** 순서로. **검증에서 100% 이상 없을 때만 적용/진행.**
- 코드 수정 후 반드시: ① 문법 컴파일(py_compile) ② 핵심 로직 단위테스트 ③ Streamlit 헤드리스 부팅(healthz 200) 확인.

## 0-1. 알려진 작업 환경 주의점 (중요)
- 이 개발 환경(샌드박스)은 **외부 통신이 막혀 있음** → 네이버/도매꾹/OpenAI/Gemini 등 **실제 API 호출 결과는 여기서 검증 불가**. 코드 경로·폴백·로직만 검증 가능. **실데이터 최종 확인은 사용자 PC에서.**
- 파일 저장 시 가끔 **한글 깨짐(예: 도매꾹→도매꽉) 또는 파일 끝 잘림(truncation)** 발생. 의심되면 백업본에서 Python으로 재구성하고 재검증할 것. (bash heredoc도 한글을 깨뜨릴 수 있으니, 한글 치환은 유니코드 코드포인트로 처리.)

## 2. 앱 개요 / 흐름 (직렬 구조)
**① 키워드 분석 → ② 소싱 → ③ 상세페이지 → ④ 이미지** (Streamlit 탭 4개)
- 사이드바: .env 키 12종 감지 표시 / LLM 선택(GPT 기본) / "실데이터 사용" 토글. 외부통신 막히면 자동 모의 폴백.

## 3. 파일 구조
- `app.py` — 메인 UI (4개 탭). 약 440줄.
- `core/config.py` — .env 로딩, 모델명 기본값.
- `core/naver_keyword.py` — 네이버 검색광고 keywordstool(검색량·경쟁도) + 쇼핑 상품수/경쟁률 + 상품화점수/등급(A≥70/B≥45/C).
- `core/sourcing.py` — 도매꾹 OpenAPI getItemList(ver=4.1) + 모의 폴백. 썸네일 URL 정규화. 반환 (rows, is_real).
- `core/copy_gpt.py` — 상세페이지 카피: generate(단순) / generate_13(13섹션) / humanize(문체정리) / product_names+check_name(상품명3안·해시태그·금지어검증, 클린룸).
- `core/image_gen.py` — Gemini(나노바나나)우선→OpenAI(gpt-image-1)폴백, remove.bg, 흰배경합성, 참조이미지편집(edit_image/expand_from_reference), **fetch_image_bytes(원격URL→PNG)**.
- `core/detail_template.py` — build_detail_html / build_detail_html_13 (13섹션 HTML, 섹션별 이미지 배치).
- `core/datalab_rank.py` — 데이터랩 분야별 인기검색어(비공식, 개인용).
- `test_apis.py` — PC용 스모크 테스트.

## 4. 구현 완료된 기능
- ① 카테고리 선택 + 키워드 입력 → 분석. 결과 표에 **체크박스(선택)** → "✅ 선택 키워드 → ② 소싱" 버튼으로 전달.
- ① "🔥 분야별 인기검색어 TOP(데이터랩)" 확장 — 체크박스+소싱전달 동일 패턴.
- ① "🏷️ 상품명 3안·해시태그" 확장 — 금지어/글자수 검증 배지(✅/⚠️).
- ② 도매꾹 소싱 후보 검색, 카드/표 보기 전환, 썸네일 이미지·상품링크 클릭.
- ② 소싱 키워드란 **직접 입력 가능**(매뉴얼).
- ③ 13섹션 감정여정 카피 + 문체정리 + AI 대표이미지(B).
- ④ 프롬프트 이미지생성, 흰배경 합성, **사진→구간 이미지 확장**(4장면) → ③ 13섹션에 자동 배치.

## 5. 가장 최근 적용 (이번 세션, 검증 완료)
**"② 소싱에서 선택한 도매꾹 대표이미지를 ③로 가져와 AI가 그 사진으로 작업"**
- ② 표/카드 선택 시 `sourcing_pick`에 `thumb`(썸네일 URL) 함께 저장.
- `image_gen.fetch_image_bytes(url)` 추가: 원격 이미지→PNG 표준화(편집 API 호환).
- ③ 상단에 **대표이미지 미리보기 + "도매꾹 대표이미지 기반으로 AI 작업(권장)" 체크박스**.
  - 켜짐: `fetch_image_bytes` → `edit_image(SCENE_PROMPTS["흰배경 메인컷"])` 로 실제 상품 유지한 채 가공.
  - 실패/썸네일 없음: **기존 텍스트 생성으로 자동 폴백** → 그래도 실패 시 자리표시. 앱 안 멈춤.
- 검증 결과: py_compile OK / 단위테스트(JPEG→PNG, 빈URL예외, 썸네일추출) PASS / SCENE 키 매칭 True / Streamlit 부팅 healthz 200, 에러 없음.
- **PC에서 최종 확인 필요**: ② 상품선택 → ③ "상세페이지 생성" 시 대표이미지 기반으로 나오는지.
- 백업: `app.py.bak_thumb`, `core/image_gen.py.bak_thumb` 존재.
